# 项目概述

本仓库用于记录个人在 **Python 编程、客户端通信协议、登录流程、数据结构、加解密算法及软件兼容性** 等方向的学习与研究成果。

仓库中的代码以学习、技术验证和经合法授权的兼容性测试为目的，不代表相关 App、平台、学校、企业或服务提供商的官方实现。

> [!IMPORTANT]
> 请仅使用本人账号、本人设备、公开测试环境或已经取得明确授权的目标进行测试。无法确认授权范围时，请勿运行相关脚本。

## 研究范围

仓库按 App 分目录保存独立研究实例，目前主要涉及：

- Python 网络请求、连接重试与会话管理；
- HTTP、WAP、SOAP 和统一认证等通信流程；
- Cookie、Token、JWT、认证票据和请求头处理；
- AES、DES、RSA、Base64、MD5 等算法的兼容性实现；
- 客户端设备参数、签名参数和响应数据结构研究；
- 经合法授权的软件兼容性、安全性和自动化测试。

不同 App 的代码、依赖、运行数据和兼容性说明相互独立。新增项目时应放入单独目录，避免无关实现相互耦合。

- Python `3.10+`；
- Windows、Linux 或 macOS；

- 建议使用独立虚拟环境运行。

```bash
python -m venv .venv
```



# 今日校园登录研究

脚本路径：

```text
今日校园/今日校园.py
```

## 已实现功能

- 获取学校租户列表；
- 根据学校名称、租户代码、学校域名或校园邮箱识别学校；
- 自动解析学校统一认证入口；
- 学校统一认证账号密码登录；
- 原生手机号短信验证码登录；
- 登录成功后读取当前账号基础资料；
- 使用交互式命令行菜单运行；
- 使用当前进程内的短生命周期会话；
- 不主动将账号、密码、短信验证码和 Cookie 持久化到本地文件。

## 运行方式

Windows：

```powershell
python ".\今日校园\今日校园.py"
```

Linux 或 macOS：

```bash
python "./今日校园/今日校园.py"
```

运行菜单：

```text
1. 学校账号密码登录并获取信息
2. 手机短信验证码登录并获取信息
0. 退出
```

## 账号输入格式

学校账号密码登录支持：

```text
学校名称::账号
学校域名\账号
校园邮箱
```

示例：

```text
某某大学::20260001
example.edu.cn\20260001
20260001@example.edu.cn
```

示例仅用于说明格式，不代表任何真实学校、账号或服务。

## 运行目录

脚本运行时会创建：

```text
今日校园/runtime/
├── temp/
└── cache/
```

这些目录只用于当前运行产生的临时文件和缓存，已由根目录 `.gitignore` 排除，不应提交到仓库。

## 已知限制

- 不同学校可能使用不同版本的 CAS、AuthServer、IAP 或自定义认证系统；
- 账号密码登录不保证兼容所有学校；
- 部分学校可能要求图形验证码、滑块验证或二次认证；
- 当前脚本不识别或绕过图形验证码和滑块验证；
- 统一认证成功后，学校业务侧会话仍可能未建立；
- 手机号未注册或未绑定学校时，短信登录无法建立完整会话；
- 服务端接口、字段、加密参数和跳转流程可能随时调整。

# 喜鹊儿登录研究

脚本路径：

```text
喜鹊儿/喜鹊儿.py
```

## 已实现功能

- 根据学校代码或学校名称查询学校服务地址；
- 支持用户手动提供学校服务地址 `serviceUrl`；
- 优先尝试 WAP `getLoginInfoNew` 登录链路；
- 在兼容性需要下尝试旧 SOAP 登录链路；
- 尝试 manager 普通表单与兼容签名登录链路；
- 对不同历史版本的账号字段、密码字段和签名格式进行单账号兼容尝试；
- 登录成功后调用 `auth/getUserInfo` 读取当前账号资料；
- 资料接口不可用时，回退到登录响应中已经获得的资料；
- 对错误摘要中的密码、Token、JWT 和签名字段进行基础脱敏；
- 所有认证状态只保存在当前 Python 进程内存中。

这里的“多链路兼容”是针对同一用户主动输入的一组账号凭据依次验证不同历史接口，不是批量账号登录、密码猜测或凭证填充功能。

## 运行方式

Windows：

```powershell
python ".\喜鹊儿\喜鹊儿.py"
```

Linux 或 macOS：

```bash
python "./喜鹊儿/喜鹊儿.py"
```

运行菜单：

```text
1. 账号密码登录并获取资料
0. 退出
```

登录时需要输入：

- 本人账号；
- 本人账号密码；
- 学校代码或学校名称，可留空；
- 学校服务地址 `serviceUrl`，可留空。

## 输出与隐私

登录成功后，终端可能输出：

- 实际成功的登录链路；
- 资料数据来源；
- 学校服务地址；
- 用户 ID、姓名和用户类型；
- 学校、学号、学院、专业、班级和入学年份；
- 手机号和邮箱。

这些字段可能包含个人信息，不得直接公开完整终端截图、录屏或日志。

## 已知限制

- 不同学校部署的服务版本和服务地址可能不同；
- 学校查询接口可能无法返回目标学校；
- 旧 WAP、SOAP 或 manager 接口可能被关闭、迁移或调整；
- 某些学校可能使用额外验证码、设备认证或二次认证；
- 脚本不保证所有兼容链路始终可用；
- 账号资料字段可能因学校和服务版本不同而缺失。

# 工学云登录研究

脚本路径：

```text
工学云/工学云.py
```

运行效果文件：

[查看工学云运行效果](./工学云/效果图.png)

## 已实现功能

- 获取 `blockPuzzle` 滑块验证所需的背景图和滑块图；
- 使用 OpenCV Canny 边缘检测和模板匹配计算缺口位置；
- 生成验证轨迹并提交滑块校验；
- 调用 `session/user/v6/login` 完成本人账号登录；
- 使用 AES/ECB/PKCS5 处理业务参数；
- 生成接口所需的 MD5 签名字段；
- 登录成功后调用 `practice/plan/v3/getPlanByStu`；
- 读取当前账号资料和本人实习计划；
- 所有认证状态只保存在当前 Python 进程内存中；
- 不包含签到、打卡、周报、月报或其他业务数据提交功能。

滑块处理仅用于当前登录流程的兼容性研究。不得将相关实现用于批量验证、规避平台限制、干扰验证服务或访问未经授权的账号。

## 运行方式

Windows：

```powershell
python ".\工学云\工学云.py"
```

Linux 或 macOS：

```bash
python "./工学云/工学云.py"
```

运行时按照终端提示输入本人账号和密码。脚本会在需要时获取并处理登录滑块验证，然后读取本人账号及实习计划信息。

## 输出与隐私

工学云脚本的数据结构可能包含：

- 用户 ID、用户名和姓名；
- 手机号；
- 学校、学院、专业和班级；
- 学号和学生 ID；
- 用户角色；
- 实习计划内容；
- Token 和过期时间。

> [!WARNING]
> 当前脚本的终端输出可能包含 Token 等会话凭据。不得公开原始终端截图、完整 JSON 输出或运行日志。分享问题信息前，必须删除账号、密码、手机号、学号、Token、用户 ID 和实习计划中的个人信息。

## 已知限制

- OpenCV 模板匹配结果可能受验证码图片版本、缩放比例和图像质量影响；
- 验证接口、登录接口、签名参数和 App 版本可能随时调整；
- 频繁请求可能触发服务端限制；
- 登录成功不代表实习计划接口一定返回完整数据；
- 当前项目属于实验性研究代码，不建议用于生产环境或无人值守运行。
