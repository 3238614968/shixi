# App 研究

本仓库用于记录个人在 **Python 编程、客户端通信协议、登录流程、数据结构及软件兼容性** 等方向的学习与研究成果。

仓库中的代码以学习、技术验证和经合法授权的测试为目的，不代表相关 App、平台、学校、企业或服务提供商的官方实现。

## 项目定位

本仓库后续可能收录不同 App 的独立研究实例，主要涉及：

- Python 网络请求与会话管理；
- HTTP 接口和客户端通信流程分析；
- Web、移动端及统一认证登录流程；
- HTML 表单、重定向、Cookie 和认证票据处理；
- AES、DES、RSA、Base64、Hash 等算法的兼容性实现；
- 客户端请求头、设备参数和响应数据结构研究；
- 经合法授权的软件兼容性、安全性和自动化测试。

每个 App 将使用独立目录存放，尽量避免不同项目之间的依赖和运行数据相互影响。

## 今日校园登录研究

当前脚本文件：

```text
今日校园/今日校园.py
```

### 已实现功能

- 获取学校租户列表；
- 根据学校名称、租户代码、学校域名或校园邮箱识别学校；
- 自动解析学校统一认证入口；
- 学校统一认证账号密码登录；
- 原生手机号短信验证码登录；
- 登录成功后读取当前账号基础资料；
- 交互式命令行菜单；
- 运行期间使用短生命周期内存会话；
- 不主动将账号、密码、短信验证码和 Cookie 持久化到本地文件。

### 明确不提供的功能

当前脚本不包含以下功能：

- 自动打卡、签到或补签；
- 日报、周报、健康信息或其他业务表单提交；
- 批量账号登录或账号池；
- Cookie、Token 或登录凭证持久化；
- 图形验证码自动识别；
- 滑块验证码绕过；
- 风控规避或设备环境对抗；
- 伪造位置、时间或业务数据；
- 付费、会员、授权或访问控制绕过。

当服务端要求图形验证码或滑块验证时，脚本会停止当前登录流程，并提示使用官方客户端完成必要验证。

## 环境要求

- Python `3.10+`
- Windows、Linux 或 macOS
- 可正常访问对应服务的网络环境

Python 依赖：

```text
requests
pycryptodome
```

## 安装与运行

### 1. 克隆仓库

```bash
git clone https://github.com/3238614968/shixi.git
cd shixi
```

### 2. 创建虚拟环境

```bash
python -m venv .venv
```

Windows：

```powershell
.venv\Scripts\activate
```

Linux 或 macOS：

```bash
source .venv/bin/activate
```

### 3. 安装依赖

```bash
python -m pip install --upgrade pip
pip install requests pycryptodome
```

### 4. 运行脚本

Windows：

```powershell
python ".\今日校园\今日校园.py"
```

Linux 或 macOS：

```bash
python "./今日校园/今日校园.py"
```

运行后可选择：

```text
1. 学校账号密码登录并获取信息
2. 手机短信验证码登录并获取信息
0. 退出
```

## 账号输入方式

学校账号密码登录支持以下输入形式：

```text
学校名称::账号
学校域名\账号
校园邮箱
```

示例：

```text
某某大学::20260001
example.edu.cn\20260001
20260001@example.edu.cn
```

示例仅用于说明输入格式，不代表任何真实学校、账号或服务。

## 数据与隐私说明

脚本通过 `requests.Session` 在当前 Python 进程内维护登录会话，设计目标是不主动将以下信息写入本地持久化文件：

- 登录账号；
- 登录密码；
- 短信验证码；
- Cookie；
- SessionToken；
- 登录票据。

需要注意：

1. 登录成功后，脚本会在终端中输出当前账号资料；
2. 终端历史、录屏软件、系统日志或第三方运行环境可能保存输出内容；
3. 不要公开包含姓名、学号、手机号、学校、组织关系或认证信息的截图；
4. 不要把真实请求日志、Cookie、Token、验证码或账号密码提交到 Issue；
5. 在共享调试信息前，应当完成脱敏处理。

建议脱敏格式：

```text
账号：2026******01
手机号：138****0000
姓名：张*
Token：已删除
Cookie：已删除
```

## 兼容性与已知限制

不同学校可能使用不同版本的 CAS、AuthServer、IAP 或自定义统一认证系统，因此：

- 账号密码登录不保证兼容所有学校；
- 部分学校可能要求图形验证码、滑块验证或二次认证；
- 统一认证成功后，学校业务侧会话仍可能未建立；
- 服务端接口、字段、加密参数和跳转流程可能随时调整；
- 手机号未注册或未绑定学校时，原生短信登录无法建立完整会话；
- 当前项目属于实验性研究代码，不建议用于生产环境。

发现兼容性问题时，可以提交经过脱敏的错误信息和最小复现步骤。
