# App 协议研究与 Python 实例

本仓库用于记录个人在 **Python 编程、客户端通信协议、登录流程、数据结构及软件兼容性** 等方向的学习与研究成果。

仓库中的代码以学习、技术验证和经合法授权的测试为目的，不代表相关 App、平台、学校、企业或服务提供商的官方实现。

> [!IMPORTANT]
> 请仅使用本人账号、本人设备、公开测试环境或已取得明确授权的目标进行测试。无法确认授权范围时，请勿运行相关脚本。

## 项目定位

本仓库按 App 分目录保存独立研究实例，主要涉及：

- Python 网络请求、连接重试与会话管理；
- HTTP、WAP、SOAP 等客户端通信流程；
- Web、移动端及统一认证登录流程；
- HTML 表单、重定向、Cookie、Token 和认证票据处理；
- AES、DES、RSA、Base64、MD5 等算法的兼容性实现；
- 客户端请求头、设备参数、签名参数和响应结构研究；
- 经合法授权的软件兼容性、安全性和自动化测试。

不同 App 的代码、运行数据和兼容性说明相互独立。新增项目时应放入新的 App 目录，不应把无关实现堆放在仓库根目录。

# 今日校园登录研究

脚本路径：

```text
今日校园/今日校园.py
```

## 已实现功能

- 获取学校租户列表；
- 根据学校名称、租户代码、学校域名或校园邮箱识别学校；
- 自动解析学校统一认证入口；
- 学校统一认证账号密码登录；
- 原生手机号短信验证码登录；
- 登录成功后读取当前账号基础资料；
- 使用交互式命令行菜单运行；
- 使用当前进程内的短生命周期会话；
- 不主动将账号、密码、短信验证码和 Cookie 持久化到本地文件。

## 运行方式

Windows：

```powershell
python ".\今日校园\今日校园.py"
```

Linux 或 macOS：

```bash
python "./今日校园/今日校园.py"
```

运行菜单：

```text
1. 学校账号密码登录并获取信息
2. 手机短信验证码登录并获取信息
0. 退出
```

## 账号输入格式

学校账号密码登录支持：

```text
学校名称::账号
学校域名\账号
校园邮箱
```

示例：

```text
某某大学::20260001
example.edu.cn\20260001
20260001@example.edu.cn
```

示例仅用于说明格式，不代表任何真实学校、账号或服务。

## 运行目录

脚本运行时会创建：

```text
今日校园/runtime/
├── temp/
└── cache/
```

这些目录只用于当前运行产生的临时文件和缓存，已由根目录 `.gitignore` 排除，不应提交到仓库。

## 已知限制

- 不同学校可能使用不同版本的 CAS、AuthServer、IAP 或自定义认证系统；
- 账号密码登录不保证兼容所有学校；
- 部分学校可能要求图形验证码、滑块验证或二次认证；
- 脚本不识别或绕过图形验证码和滑块验证；
- 统一认证成功后，学校业务侧会话仍可能未建立；
- 手机号未注册或未绑定学校时，短信登录无法建立完整会话；
- 服务端接口、字段、加密参数和跳转流程可能随时调整。

# 喜鹊儿登录研究

脚本路径：

```text
喜鹊儿/喜鹊儿.py
```

## 已实现功能

- 根据学校代码或学校名称查询学校服务地址；
- 支持用户手动提供学校服务地址 `serviceUrl`；
- 优先尝试 WAP `getLoginInfoNew` 登录链路；
- 在兼容性需要下尝试旧 SOAP 登录链路；
- 尝试 manager 普通表单与兼容签名登录链路；
- 对不同历史版本的账号字段、密码字段和签名格式进行单账号兼容尝试；
- 登录成功后调用 `auth/getUserInfo` 读取当前账号资料；
- 资料接口不可用时，回退到登录响应中已经获得的账号资料；
- 对错误摘要中的密码、Token、JWT、签名等字段进行基础脱敏；
- 所有认证状态只保存在当前 Python 进程内存中。

这里的“多链路兼容”是针对同一用户主动输入的一组账号凭据依次验证不同历史接口，不是批量账号登录、密码猜测或凭证填充功能。

## 运行方式

Windows：

```powershell
python ".\喜鹊儿\喜鹊儿.py"
```

Linux 或 macOS：

```bash
python "./喜鹊儿/喜鹊儿.py"
```

运行菜单：

```text
1. 账号密码登录并获取资料
0. 退出
```

登录时需要输入：

- 本人账号；
- 本人账号密码；
- 学校代码或学校名称，可留空；
- 学校服务地址 `serviceUrl`，可留空。

学校代码、学校名称和服务地址用于学校服务发现及兼容性匹配。脚本不会把输入的密码写入本地配置文件。

## 输出内容

登录成功后，终端可能输出：

- 实际成功的登录链路；
- 资料数据来源；
- 学校服务地址；
- 用户 ID；
- 姓名；
- 用户类型；
- 学校和学号；
- 性别、学院、专业、班级及入学年份；
- 手机号和邮箱。

这些字段可能包含个人信息。不得直接公开完整终端截图、录屏或日志。

## 已知限制

- 不同学校部署的服务版本和服务地址可能不同；
- 学校查询接口可能无法返回目标学校；
- 旧 WAP、SOAP 或 manager 接口可能被关闭、迁移或调整；
- 某些学校可能使用额外验证码、设备认证或二次认证；
- 脚本不保证所有兼容链路始终可用；
- 账号资料字段可能因学校和服务版本不同而缺失；
- 项目属于协议兼容性研究代码，不建议用于生产环境。

脚本通过 `requests.Session` 在当前 Python 进程中维护临时登录状态，设计目标是不主动将以下信息持久化到本地：

- 登录账号；
- 登录密码；
- 短信验证码；
- Cookie；
- Token、JWT 和 Session；
- 登录票据和签名材料。

仍需注意：

1. 登录成功后，脚本会在终端输出当前账号资料；
2. 终端历史、Shell 记录、录屏软件、系统日志或第三方运行环境可能保存输出；
3. 学号、手机号、邮箱、学院、班级等字段属于个人信息；
4. 不要把真实请求日志、Cookie、Token、验证码或账号密码提交到 Issue；
5. 在共享错误信息前，应完成脱敏处理；
6. 公共电脑或不可信运行环境不适合输入真实账号凭据。
